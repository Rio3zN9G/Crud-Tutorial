Apa Itu CRUD?

Memahami konsep dasar operasi database

CREATE

Operasi untuk membuat/menambahkan data baru ke dalam database. Dalam konteks aplikasi, ini biasanya berupa form input yang akan menyimpan data baru.

READ

Operasi untuk membaca/mengambil data dari database. Ini bisa berupa menampilkan daftar data atau menampilkan detail dari satu data tertentu.

UPDATE

Operasi untuk memperbarui data yang sudah ada di database. Biasanya melibatkan form pengeditan yang diisi dengan data lama sebelum diubah.

DELETE

Operasi untuk menghapus data dari database. Biasanya memerlukan konfirmasi sebelum eksekusi karena data yang terhapus tidak bisa dikembalikan.

Fungsi dan Kegunaan CRUD

Mengapa CRUD sangat penting dalam pengembangan aplikasi?

Manajemen Data

CRUD menyediakan fondasi untuk semua operasi manajemen data dalam aplikasi, memungkinkan pengguna untuk berinteraksi dengan data secara efektif.

Interaksi Aplikasi

Hampir semua fitur dalam aplikasi web modern melibatkan operasi CRUD, dari posting media sosial hingga transaksi e-commerce.

Standard Industri

CRUD merupakan standar industri untuk operasi database, didukung oleh semua sistem manajemen database modern seperti MySQL, PostgreSQL, MongoDB.

Contoh Penerapan CRUD
  • Aplikasi To-Do List (menambah, melihat, mengedit, menghapus tugas)
  • Sistem Manajemen Konten (posting blog, produk, dll)
  • Aplikasi E-commerce (manajemen produk, keranjang belanja)
  • Sistem Manajemen User (registrasi, login, profil)

JavaScript CRUD dengan Fetch API
// CREATE - Tambah data baru
async function createItem(itemData) {
    const response = await fetch('/api/items', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
        },
        body: JSON.stringify(itemData),
    });
    return await response.json();
}

// READ - Ambil semua data
async function getItems() {
    const response = await fetch('/api/items');
    return await response.json();
}

// READ - Ambil data spesifik
async function getItem(id) {
    const response = await fetch(`/api/items/${id}`);
    return await response.json();
}

// UPDATE - Perbarui data
async function updateItem(id, updatedData) {
    const response = await fetch(`/api/items/${id}`, {
        method: 'PUT',
        headers: {
            'Content-Type': 'application/json',
        },
        body: JSON.stringify(updatedData),
    });
    return await response.json();
}

// DELETE - Hapus data
async function deleteItem(id) {
    const response = await fetch(`/api/items/${id}`, {
        method: 'DELETE',
    });
    return await response.json();
}
Node.js Express CRUD Endpoints
const express = require('express');
const router = express.Router();
const Item = require('../models/Item');

// CREATE - Tambah item baru
router.post('/', async (req, res) => {
    try {
        const newItem = new Item(req.body);
        const savedItem = await newItem.save();
        res.status(201).json(savedItem);
    } catch (err) {
        res.status(400).json({ message: err.message });
    }
});

// READ - Dapatkan semua item
router.get('/', async (req, res) => {
    try {
        const items = await Item.find();
        res.json(items);
    } catch (err) {
        res.status(500).json({ message: err.message });
    }
});

// READ - Dapatkan item spesifik
router.get('/:id', async (req, res) => {
    try {
        const item = await Item.findById(req.params.id);
        if (!item) return res.status(404).json({ message: 'Item tidak ditemukan' });
        res.json(item);
    } catch (err) {
        res.status(500).json({ message: err.message });
    }
});

// UPDATE - Perbarui item
router.put('/:id', async (req, res) => {
    try {
        const updatedItem = await Item.findByIdAndUpdate(
            req.params.id,
            req.body,
            { new: true }
        );
        if (!updatedItem) return res.status(404).json({ message: 'Item tidak ditemukan' });
        res.json(updatedItem);
    } catch (err) {
        res.status(400).json({ message: err.message });
    }
});

// DELETE - Hapus item
router.delete('/:id', async (req, res) => {
    try {
        const deletedItem = await Item.findByIdAndDelete(req.params.id);
        if (!deletedItem) return res.status(404).json({ message: 'Item tidak ditemukan' });
        res.json({ message: 'Item berhasil dihapus' });
    } catch (err) {
        res.status(500).json({ message: err.message });
    }
});

module.exports = router;
SQL CRUD Operations
-- CREATE TABLE
CREATE TABLE items (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- CREATE - Tambah data baru
INSERT INTO items (name, description) 
VALUES ('Item 1', 'Deskripsi untuk item pertama');

-- READ - Ambil semua data
SELECT * FROM items;

-- READ - Ambil data spesifik
SELECT * FROM items WHERE id = 1;

-- UPDATE - Perbarui data
UPDATE items 
SET name = 'Item Terupdate', description = 'Deskripsi baru'
WHERE id = 1;

-- DELETE - Hapus data
DELETE FROM items WHERE id = 1;

Pertanyaan Umum

Hal-hal yang sering ditanyakan tentang CRUD

CRUD adalah konsep operasi dasar database (Create, Read, Update, Delete), sedangkan REST API adalah arsitektur untuk membangun web services yang sering menggunakan operasi CRUD. REST API memetakan operasi CRUD ke HTTP methods (POST, GET, PUT/PATCH, DELETE).

Tidak, konsep CRUD berlaku untuk berbagai jenis database termasuk relasional (MySQL, PostgreSQL), NoSQL (MongoDB, Firebase), bahkan untuk penyimpanan lokal seperti localStorage di browser.

Validasi dapat dilakukan di beberapa level:

  • Frontend: Validasi form sebelum data dikirim ke server
  • Backend: Validasi data sebelum diproses ke database
  • Database: Constraints dan schema validation

Contoh validasi backend di Express.js bisa menggunakan library seperti Joi atau express-validator.
